História dos computadores

Os computadores evoluiram muito ao longo dos anos. No inicio eram super maquinas pesadas no qual eram impossiveis de serem transportadas e facilmente implantada.
Pense em uma sala gigante com uma máquina enorme para equações aritmedicas e criptrografia de dados. Posteriormente tivemos avanços até chegarmos nos computadores atuais.

O que são Clients?

Clients nada mais são de dispositivos que requisitam informações e dados de um servidor, ou seja são responsaveis por interagir atraves de um navegador ou aplicativo.
Podem ser eles: Celulares, Computadores, Tablets, os proprios navegadores e qualquer dispositivo que solicite dados á um banco de dados/servidor.

O que são Servers?

Servers são servidores, ao contrario dos clientes não solicitam informações, eles são responsaveis pela resposta a essas solicitações. Podem ser atraves de links, protocolos (http, ftp, dns...) ou simplesmente um carregamento da pagina ou sessão de login.
Servidores estão geralmente em grandes datacenters com muitos computadores, no qual são mais potentes dos habituais.

Como é uma estrutura básica de um HTML?

Estrutura básica de uma HTML é formada por "Tags" e a forma mínima padrão de toda página é da seguinte forma:

< html >
< head >nome do site< / head >
< body >

< /body >
< /html >

A partir dessa estrutura colocamos mais tags necessárias para manter um html mais robusto e estruturado!
Outras tags bastante utilizadas:
< i >< /i > Italico, < strong >< /strong > negrito, < input > cria um campo para inserismos dados, < h1 >< /h1 > Titulos, < h2 >< /h2 > subtitulos, < p >< /p > paragrafos de textos, entre outros.

Listas ordenadas e não ordenadas

Aqui tempos 2 tipos de listas em html. De fato as diferenças são apenas de visualização.
Na lista ordenada usamos a tag < ol > para abrir a lista e < /ol > para fecha-la.
< ol >
< li > item 1 < /li >
< li > item 2 < /li >
< /ol >

Ficará assim:

  1. item 1
  2. item 2

Na lista não ordenada usamos a tag < ul > para abrir a lista e < /ul > para fecha-la. Mantendo a mesma tag para as linhas: < li >< /li >.
< ul >
< li > item 1 < /li >
< li > item 2 < /li >
< /ul >

Ficará assim:
  • item 1
  • item 2
